# Forget Skill

ユーザーが「忘れて」「削除して」「記憶を消して」と言った時に、指定された記憶を削除します。

## 削除手順

1. **対象の特定**: ユーザーの指定に基づき削除対象を特定
2. **確認**: 削除前に必ず確認を取る
3. **削除実行**: ファイルを削除
4. **index.md 更新**: エントリを削除

## 確認ダイアログ

```
🗑️ 以下の記憶を削除しますか？

**API設計方針** (2026-01-22)
- RESTful API の設計ルール

[y/n] で回答してください。
```

## 一括削除

### 日付指定
```
「1週間前より古い記憶を削除して」
→ 対象ファイル一覧を提示 → 確認 → 削除
```

### タグ指定
```
「#draft タグの記憶をすべて削除して」
→ 対象ファイル一覧を提示 → 確認 → 削除
```

## 重要

- **確認なしに削除しない**
- 複数ファイルの場合は一覧を提示
- 削除後は復元不可であることを伝える

## 応答例

```
✅ 削除しました: `.memories/2026-01-22_api-design.md`

index.md も更新しました。
```
